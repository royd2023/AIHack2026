# BuckeyePathfinder

**AI Career Intelligence Platform** — Built for the IBM AI Skills Builder Hackathon at The Ohio State University.

BuckeyePathfinder helps OSU CSE students make smarter course decisions by analyzing the real job market, understanding what courses actually teach, and generating personalized, AI-optimized academic plans.

---

## Architecture

```
raw_data/                         ← Real data collected by team
  job_postings.json               ← 30-50 real postings (LinkedIn, Indeed, Glassdoor)
  osu_courses.json                ← 35-45 OSU courses (syllabi.engineering.osu.edu)
  course_reviews.json             ← Optional: Rate My Professor reviews

backend/
  pipelines/
    watsonx_client.py             ← Shared IBM watsonx.ai client (Granite LLM + Embeddings)
    skill_extractor.py            ← STAGE 1: job postings → skill demand index via Granite
    course_fingerprinter.py       ← STAGE 2: syllabi → course skill vectors + similarity
    gap_analyzer.py               ← STAGE 3: student state → gap analysis → 3 plans
  api/
    routes.py                     ← FastAPI routes
  main.py                         ← FastAPI app + CORS
  data/
    skill_index.json              ← Generated by Stage 1
    course_fingerprints.json      ← Generated by Stage 2

frontend/
  src/
    App.jsx                       ← Main app + routing
    components/
      LoadingOverlay.jsx          ← Animated 3-step Granite progress indicator
      CourseSelector.jsx          ← Searchable multi-select course picker
      SkillRadar.jsx              ← Recharts radar chart (your skills vs career target)
      GapCards.jsx                ← Top skill gaps with demand bars + company attribution
      PlanCards.jsx               ← 3 plan variants with semester accordions
      OverlapWarnings.jsx         ← Course redundancy warnings from embeddings
      HowItWorks.jsx              ← Pipeline transparency section
    utils/
      api.js                      ← API client
```

## 3-Stage AI Pipeline

```
Job Postings → [IBM Granite LLM] → Skill Demand Index
                                           ↓
OSU Syllabi  → [IBM Granite LLM + Granite Embeddings] → Course Fingerprints + Similarity
                                           ↓
Student Input → [Gap Scoring + IBM Granite LLM] → 3 Optimized Course Plans
```

### Stage 1: Skill Extraction
- Reads `raw_data/job_postings.json`
- Calls `ibm/granite-3-3-8b-instruct` to extract exact skills per posting
- Aggregates demand scores, co-occurrence, and company attribution
- Outputs `backend/data/skill_index.json`

### Stage 2: Course Fingerprinting
- Reads `raw_data/osu_courses.json` (with real learning outcomes)
- Granite extracts skills taught per course, cross-referenced against industry demand
- `ibm/granite-embedding-278m-multilingual` generates course embedding vectors
- Cosine similarity matrix detects redundant course pairs (>0.65 threshold)
- Outputs `backend/data/course_fingerprints.json`

### Stage 3: Gap Analysis + Planning
- Computes student's current skill state from completed courses
- Scores gaps: `gap_score = demand_score × (1 − student_proficiency)`
- Ranks available courses by gap-closure value (respecting prerequisites)
- Calls Granite to generate **3 Pareto-optimal plans**:
  - 🚀 Fastest to Job-Ready
  - 🔀 Maximum Optionality
  - ⚖️ Balanced Growth

## Setup

### Prerequisites
- Python 3.11+
- Node.js 18+
- IBM Cloud account with watsonx.ai project

### 1. Clone & Setup

```bash
bash setup.sh
```

### 2. Configure credentials

```bash
cp .env.example .env
```

Edit `.env`:
```
WATSONX_API_KEY=your_ibm_cloud_api_key
WATSONX_PROJECT_ID=your_project_id
WATSONX_URL=https://us-south.ml.cloud.ibm.com
```

### 3. Add real data

Place your collected data in `raw_data/`:
- `raw_data/job_postings.json`
- `raw_data/osu_courses.json`
- `raw_data/course_reviews.json` (optional)

### 4. Run the pipeline (once)

```bash
# Start backend first
source venv/bin/activate
python -m uvicorn backend.main:app --reload --port 8000

# Trigger pipeline
curl -X POST http://localhost:8000/api/run-pipeline
```

### 5. Start frontend

```bash
cd frontend && npm run dev
```

Open **http://localhost:5173**
